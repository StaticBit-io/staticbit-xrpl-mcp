# Соглашение о двуязычной документации

Проект поставляет документацию на **двух языках** — английском (канонический источник по умолчанию) и русском (точное зеркало). Раскладка унифицирована для **всех** репозиториев StaticBit MCP и проверяется в CI скриптом `scripts/check-translations.sh` (workflow `.github/workflows/docs-i18n.yml`).

## Раскладка файлов

Есть два правила размещения — в зависимости от того, лежит ли файл внутри `docs/`.

### Внутри `docs/` — зеркальное поддерево

Английская версия канонична по своему естественному пути; русский перевод зеркалит тот же относительный путь под `docs/ru/`:

```
docs/features.md                 # English (канонический)
docs/ru/features.md              # Russian mirror

docs/examples/amm-clawback.md    # English (канонический)
docs/ru/examples/amm-clawback.md # Russian mirror
```

То есть `docs/<path>.md` ⇄ `docs/ru/<path>.md`. Поддерево `docs/ru/` содержит **только** переводы — у каждого файла в нём должен быть английский источник по соответствующему пути `docs/<path>.md`, и у каждой английской страницы под `docs/` (вне `docs/ru/`) должно быть зеркало.

### Вне `docs/` — суффикс-сиблинг

Для файлов в корне репозитория и под `plugins/*`, `examples/*`, `servers/*`, `infra/` русский перевод — это сиблинг `.ru.md` рядом с английским `.md`:

```
README.md                        # English (канонический)
README.ru.md                     # Russian sibling

plugins/xrpl-cloud/README.md     # English (канонический)
plugins/xrpl-cloud/README.ru.md  # Russian sibling
```

То есть `<dir>/X.md` ⇄ `<dir>/X.ru.md`.

**Английская версия — source-of-truth** для перекрёстных ссылок из кода, integration-тестов и commit-сообщений CI: именно её первой видит международная аудитория.

## Баннер переключения языка

Каждая страница начинается с однострочного баннера, указывающего на свою пару. Размещайте его **первой строкой** файла, перед заголовком H1.

**На английских страницах:**
```markdown
>  🌐 **Language**: **English** | [Русский](<relative-path-to-russian>)
```

**На русских страницах:**
```markdown
>  🌐 **Язык**: [English](<relative-path-to-english>) | **Русский**
```

Относительный путь зависит от глубины. Для страницы под `docs/` русское зеркало лежит на уровень глубже, например из `docs/features.md` это `ru/features.md`; из `docs/ru/features.md` английский источник — `../features.md`. Для `docs/ru/examples/x.md` английский источник — `../../examples/x.md`. Для суффикс-пар пара лежит сиблингом в той же директории (`README.md` ⇄ `README.ru.md`).

## Что переводить / что оставлять как есть

- **Тело документа**: полный перевод. Обе версии должны быть функционально эквивалентны — те же секции, те же таблицы, те же сниппеты кода.
- **Блоки кода / команды / пути / URL / config-ключи**: **байт-в-байт идентичны** между языками — они ссылаются на одни и те же XRPL/MCP-сущности.
- **Имена tool'ов** (например `xrpl_payment_prepare`) и **engine result codes** (`tecNO_PERMISSION`): никогда не переводятся.
- **Markdown-ссылки на другие страницы**: пересчитывайте относительные пути под расположение файла. Из `docs/ru/<x>.md` сиблинг-перевод — `<y>.md` (то же поддерево `docs/ru/`), английский источник — `../<x>.md`, файл в корне репозитория — `../../<file>`.
- **Комментарии в коде**: остаются на английском (согласно глобальным правилам стиля C#).

## CI-гейт

`scripts/check-translations.sh` (локально: `bash scripts/check-translations.sh`) проверяет, что:

1. У каждого `docs/**/*.md` (кроме `docs/ru/`) есть зеркало `docs/ru/<rel>.md`, а у каждого `docs/ru/**/*.md` есть английский источник — без сирот.
2. У каждого `X.md` в суффикс-зоне есть сиблинг `X.ru.md`.
3. Каждая русская пара — **не-заглушка** (≥ 200 байт).

### `.i18nignore`

Английские-только, генерируемые или agent-instruction файлы (например `CHANGELOG.md`, генерируемые каталоги, agent-файлы `CLAUDE.md`) исключаются перечислением их относительного пути в корневом `.i18nignore` (по одному пути на строку; пустые строки и `#`-комментарии допустимы). **Не** переводите agent-instruction файлы `CLAUDE.md` — вместо этого исключайте их.

### `.i18npairs` (опционально)

Для документов, лежащих вне конвенциональных мест, можно добавить явное соответствие `en-path:ru-path` в корневой файл `.i18npairs` (по одной паре на строку; `#`-комментарии допустимы).

## Добавление нового документа

1. Сначала пишите английский файл (`docs/<path>.md` или `<dir>/X.md`).
2. Добавьте баннер переключения языка в начало.
3. Создайте русскую пару: `docs/ru/<path>.md` для docs или `<dir>/X.ru.md` для суффикс-зоны.
4. Ссылайтесь на **английское** имя файла из родительского TOC / индекса.
5. Запускайте `bash scripts/check-translations.sh`, пока не выведется `OK: bilingual docs check passed.`

## Вне области

- **XML doc-comments в коде**: остаются только на английском (соответствует глобальным правилам стиля C#).
- **Commit-сообщения / описания PR / issue-шаблоны**: на английском.
- **Генерируемые файлы и agent-инструкции `CLAUDE.md`**: не переводятся — исключаются через `.i18nignore`.
